# Screenshot Drop

A browser extension that captures a screenshot, lets you annotate it, and
POSTs the result to a small Go **receiving service** running on another
machine on your LAN. The service saves the file and hands back its absolute
path, which the extension copies to your clipboard — so you can paste a
real, local file path straight into Claude Code running on that machine,
even when you're working over a remote session and can't paste an image
directly.

## Repo layout

```
extension/   TypeScript + esbuild, built for both Brave/Chromium (MV3) and Firefox (MV3)
service/     Go, stdlib only, compiles to a single static binary (snapdropd)
```

Inside `extension/src/`:

- **`core/`** — framework-free capture/annotation logic (geometry, annotation
  state, the canvas renderer, the `AnnotationEditor`, PNG export, full-page
  stitching). No browser-extension APIs, so this layer can be reused as-is
  inside a future Wails desktop app without a rewrite.
- **`platform/`** — small interfaces (`Transport`, `DestinationStore`) plus
  their real implementations (`fetch`-based HTTP upload, `browser.storage`).
- **`ext/`** — the actual extension glue: background service worker, popup,
  content script, and the annotate/options pages, all wired on top of
  `core/` and `platform/` via [webextension-polyfill](https://github.com/mozilla/webextension-polyfill).

## Quick start — Computer B (the receiving machine, Ubuntu)

```bash
make -C service build          # produces service/bin/snapdropd (static binary)
service/bin/snapdropd -gen-token   # prints a token — save it, you'll need it below
```

For running it permanently under systemd (unit file, env file, directory
setup, firewall notes), follow **[service/deploy/install.md](service/deploy/install.md)**
— don't duplicate that here.

## Quick start — Computer A (the browser)

```bash
cd extension
npm install
npm run build     # builds both extension/dist/chromium and extension/dist/firefox
```

**Brave / Chromium:**
1. Open `brave://extensions`, enable Developer Mode.
2. "Load unpacked" → select `extension/dist/chromium`.

**Firefox:**
1. Open `about:debugging#/runtime/this-firefox`.
2. "Load Temporary Add-on" → select any file inside `extension/dist/firefox`
   (e.g. `manifest.json`).
3. Note: Firefox's "Load Temporary Add-on" does not persist across browser
   restarts — you'll need to reload it each time Firefox restarts during
   development. (A signed, permanently-installed build is a later step, not
   covered here.)

**Add a destination:** open the extension's Options page and add one row —
a name, the receiving service's address (`http://<computer-b-lan-ip>:9922`),
and the token from `-gen-token` above.

## Usage

1. Click the toolbar icon, pick a capture mode: **Full page**, **Visible
   tab**, or **Marked area** (drag a rectangle).
2. The capture opens in the annotate view. Draw with arrow, rectangle,
   line, or freehand pen — keys `1`-`4` switch tools, `c` toggles between
   the two colors. Undo and Clear are toolbar buttons.
3. Pick a destination (the last one used is pre-selected), add an optional
   short name, and click **Save**.
4. The receiving service's returned absolute path is copied to your
   clipboard and shown on screen — paste it into Claude Code on that
   machine.

## Security model

- **LAN only.** The receiving service is meant to bind to a LAN interface
  and never be exposed publicly — there is no TLS, by design, since it's
  assumed to run on a closed network.
- **Shared token required.** Every request to `/upload` must carry the
  token generated by `-gen-token`; requests without a matching token are
  rejected.
- **Origin lockdown.** The service only accepts requests whose `Origin` is
  a `chrome-extension://` or `moz-extension://` scheme (or no `Origin` at
  all) — an ordinary web page open in your browser cannot reach it.
- Never put this service behind a public DNS record or tunnel.

make test          # extension vitest suite + go test, from the repo root
